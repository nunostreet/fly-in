O que e um grafo?

Um grafo e uma estrutura que representa elementos ligados entre si.

  • Cada Hub e um no.
  • Cada Connection e uma ligacao entre nos.

No teu projeto, o mapa pode ser visto como um grafo onde queremos navegar do start_hub ate ao end_hub.

Objetivo desta fase

O objetivo agora nao e simular drones ainda. O objetivo e transformar o World numa estrutura de navegacao e conseguir encontrar caminhos validos.

Checklist principal

  • [ ] Converter o World num grafo.
  • [ ] Conseguir listar os vizinhos de cada hub.
  • [ ] Garantir que o start_hub_name e o end_hub_name existem no grafo.
  • [ ] Implementar uma busca de caminho simples.
  • [ ] Devolver um caminho como lista de hubs.
  • [ ] Testar com os mapas easy.
  • [ ] Validar o caso em que nao existe caminho.

Fase 1: Construir o grafo

  • [ ] Criar uma adjacency list, por exemplo dict[str, list[str]].
  • [ ] Inicializar uma entrada para cada hub do mundo.
  • [ ] Percorrer world.connections.
  • [ ] Adicionar target aos vizinhos de source.
  • [ ] Adicionar source aos vizinhos de target.
  • [ ] Evitar duplicados nas listas de vizinhos, se necessario.

Fase 2: Encontrar um caminho

  • [ ] Escolher um algoritmo simples, idealmente BFS.
  • [ ] Comecar no start_hub_name.
  • [ ] Explorar os vizinhos nivel a nivel.
  • [ ] Marcar hubs visitados.
  • [ ] Guardar o hub anterior para reconstruir o caminho.
  • [ ] Parar quando encontrares o end_hub_name.
  • [ ] Reconstruir o caminho final do fim para o inicio.
