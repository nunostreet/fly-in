Fly-in Study Notes

Como reconstruir o --capacity-info

Este guia é para fazeres a feature de memória durante a avaliação. A ideia é manter tudo simples: a flag só acrescenta linhas extra ao output normal, uma para hubs e outra para connections.

Objetivo Mental

Regra simples: no fim de cada turno, imprimir:
1. ocupação de hubs usando drone.current_hub
2. uso de connections usando o link_usage desse turno

Não precisas de mexer na lógica da simulação. Só estás a aproveitar estado que já existe.

Passo 1: Adicionar a Flag em main.py

Acrescenta a flag no argparse:

parser.add_argument("--capacity-info", action="store_true")

Depois passa esse valor ao engine:

result = SimulationEngine(world, args.capacity_info).run()

Passo 2: Guardar a Flag no Engine

Em SimulationEngine.__init__, muda a assinatura:

def __init__(self, world: World, capacity_info: bool = False) -> None:

E guarda a flag:

self._capacity_info = capacity_info
Usa False por omissão para não partir o render, que normalmente cria SimulationEngine(world).

Passo 3: Aproveitar o link_usage do Turno

No teu _run_turn(), já existe:

link_usage: dict[tuple[str, str], int] = {}

Isso já é exatamente o que precisas para as connections.

Então, no fim de _run_turn(), devolve também uma cópia:

return moves, made_progress, TurnTracker(...), link_usage.copy()

E ajusta a assinatura da função para refletir esse quarto valor.

Passo 4: Receber o link_usage no run()

Onde chamas _run_turn(), passa a receber quatro valores:

turn_moves, made_progress, turn_arr_or_dep, link_usage = self._run_turn(
    drones, path
)

Passo 5: Acrescentar as Linhas Extra Só Quando a Flag Está Ativa

No bloco onde hoje fazes:

lines.append(" ".join(turn_moves))

acrescenta:

if self._capacity_info:
    lines.extend(self._add_capacity_info(drones, link_usage))

Uso extend porque a helper vai devolver duas linhas.

Passo 6: Criar a Helper _add_capacity_info(...)

A helper recebe:

  • drones para calcular hubs
  • link_usage para calcular connections

Estrutura recomendada:

def _add_capacity_info(
        self,
        drones: list[Drone],
        link_usage: dict[tuple[str, str], int],
        ) -> list[str]:
    hub_info: list[str] = []
    for hub_name in self._world.hubs:
        used_capacity = sum(
            1 for drone in drones if drone.current_hub == hub_name
        )
        max_capacity = self._hub_capacity(hub_name)
        hub_info.append(f"{hub_name}: {used_capacity}/{max_capacity} ")

    connection_info: list[str] = []
    for connection in self._world.connections:
        key = self._connection_key(connection.source, connection.target)
        used_capacity = link_usage.get(key, 0)
        connection_info.append(
            f"{connection.source}-{connection.target}: "
            f"{used_capacity}/{connection.max_link_capacity} "
        )

    return ["".join(hub_info), "".join(connection_info)]

Passo 7: Porque Esta Solução É Fácil de Defender

Hubs

“A ocupação do hub é calculada vendo onde cada drone acabou o turno.”

Connections

“O uso da connection vem diretamente do contador link_usage do turno.”

Vantagem

Não precisas de criar estado artificial. Só reaproveitas dados que já existem na simulação.

Checklist Rápida

  1. Adicionar --capacity-info em main.py.
  2. Passar a flag para SimulationEngine(world, args.capacity_info).
  3. Guardar self._capacity_info no construtor.
  4. Fazer _run_turn() devolver também link_usage.copy().
  5. Receber esse valor no run().
  6. Chamar _add_capacity_info(drones, link_usage) quando a flag está ativa.
  7. Criar uma linha para hubs e uma linha para connections.

Teste Final

python3 main.py maps/test/testing_restricted.txt --capacity-info

Se estiver tudo bem, por turno deves ver:

  • uma linha normal de movimentos
  • uma linha com hubs
  • uma linha com connections
